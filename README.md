# ระบบวางแผนการผลิต (MRP) — ออเดอร์

เว็บแอปวางแผนการผลิตแบบไฟล์เดียว (static) ทำงานฝั่งเบราว์เซอร์ เหมาะกับ **GitHub Pages**
รองรับเก็บข้อมูลบน **Supabase** (sync หลายเครื่อง/หลายคน) หรือใช้ localStorage ในเครื่องก็ได้

โยนออเดอร์เข้าไป แล้วระบบคำนวณให้:

- 🗓️ **ส่งได้วันไหน** ต่อออเดอร์ (สถานะ ทัน/ช้ากี่วัน)
- 🧪 **วัตถุดิบขาดเท่าไร** (ต้องใช้ / มีอยู่ / ขาด) จากสูตร BOM
- 📦 **ของมีเท่าไร / ต้องผลิตเท่าไร** ต่อสินค้า
- 🚀 **เร่งส่งต้องเพิ่มกี่คน** เพื่อให้ทันกำหนด
- 📅 **แผนผลิตรายวัน** แต่ละวันต้องผลิตอะไรเท่าไร

## 📆 ปฏิทินส่งของ (แท็บหลัก)

วางแผนแบบลากวาง — 1 การ์ด = 1 พาร์ทเนอร์ (รวมทุกสินค้าที่ส่งวันเดียวกัน สีเดียวต่อพาร์ทเนอร์)

มี **2 ชนิดการ์ด** ในแต่ละวัน — ลากได้ทั้งคู่ (เดสก์ท็อป) หรือ **แตะการ์ด → แตะวัน** (แท็บเล็ต/มือถือ):
- 🚚 **การ์ดจัดส่ง** (สีทึบ ต่อพาร์ทเนอร์) — วันที่จะส่งของ
- 🏭 **การ์ดผลิต** (เส้นประเหลือง ต่อสินค้า) — **คุมเอง**: เพิ่ม/แก้จำนวน/ลบ/ลากย้ายวันได้เอง

- **แนะนำแผนผลิตอัตโนมัติ** — ปุ่ม “✨ แนะนำแผนผลิต” สร้างการ์ดผลิตแบบผลิตช้าสุดที่ยังทันให้เป็นจุดตั้งต้น แล้วปรับเองต่อได้ · “ล้างแผนผลิต” เพื่อเริ่มใหม่
- **สต็อกไหลรายวัน** = สต็อกเริ่มต้น + การ์ดผลิต − จัดส่ง (คำนวณเรียงตามวัน) — ย้ายการ์ดใบไหน ทุกวันถัดไปอัปเดตทันที
- **เตือน (ไม่บล็อก)**: ถ้าสต็อกสินค้าติดลบวันไหน (ผลิตไม่พอ) ช่องวันนั้นขึ้นขอบแดง + บอก “ต้องผลิตเพิ่ม” · ถ้าการ์ดผลิตใบไหนเกินกำลังผลิต/วัน การ์ดขึ้นแดง “เกินกำลัง”
- **คลิกวัน** เพื่อดู/แก้: การจัดส่ง (พาร์ทเนอร์) · การ์ดผลิต (เพิ่ม/แก้จำนวน/ลบ) · คงคลังสิ้นวันต่อสินค้า · วัตถุดิบที่ใช้

## โมเดลการคำนวณ
- กำลังผลิต/วัน = `ชิ้นต่อรอบ × รอบต่อคนต่อวัน × จำนวนคน` (ตามรอบ/Cycle)
- ต้องผลิต (net) = `ยอดสั่งรวม − สต็อกที่มี`
- ผลิตวันละเท่ากำลังผลิต จนครบ → ได้วันเสร็จ/วันส่ง
- เพิ่มคน = เพิ่มกำลังผลิต → ใช้คำนวณ "เร่งส่ง"

## ตั้งค่า Supabase (ทางเลือก แต่แนะนำ)
1. เปิด Supabase project → เมนู **SQL Editor** → รัน SQL นี้ 1 ครั้ง:
   ```sql
   create table if not exists mrp_state (
     id text primary key,
     data jsonb not null default '{}'::jsonb,
     updated_at timestamptz default now()
   );
   alter table mrp_state enable row level security;
   create policy "mrp anon read"   on mrp_state for select using (true);
   create policy "mrp anon write"  on mrp_state for insert with check (true);
   create policy "mrp anon update" on mrp_state for update using (true) with check (true);
   ```
2. ไปที่ **Project Settings → API** คัดลอก **Project URL** และ **anon public key**
3. เปิดแอป → แท็บ **ตั้งค่า & ข้อมูล** → วาง URL + anon key → กด **เชื่อมต่อ & ซิงก์**

> ⚠️ ใช้ **anon key เท่านั้น** (ไม่ใช่ service_role) · key เก็บในเบราว์เซอร์ ไม่ถูก commit ขึ้น repo
> นโยบาย RLS ด้านบนเปิดให้ทุกคนที่มี URL+key อ่าน/เขียนได้ — ถ้าต้องการจำกัดสิทธิ์ เพิ่ม Supabase Auth ภายหลังได้

## Deploy ขึ้น GitHub Pages
1. อัปโหลด `index.html` ไว้ที่ root ของ repo (ผ่านหน้าเว็บ GitHub หรือ git push)
2. repo → **Settings → Pages → Source: Deploy from a branch** → เลือก `main` / `(root)` → Save
3. เว็บจะขึ้นที่ `https://<username>.github.io/<repo>/`

## 🔗 เชื่อมกับ Partner POS (khokai-partner-pos)

ดึงออเดอร์ที่สร้างใน **Partner POS** (khokai-partner-pos) เข้ามาวางแผนใน MRP แล้วเขียน **วันจัดส่งที่ทำได้จริง** กลับไปที่ Partner POS โดยอัตโนมัติ — ผ่าน Supabase โปรเจกต์เดียวกัน (แบบเดียวกับ khokai-oms)

**ต้องเชื่อม Supabase โปรเจกต์เดียวกับ Partner POS ก่อน** (แท็บ *ตั้งค่า & ข้อมูล*) — ใช้ anon key เดียวกัน แอปจะอ่าน/เขียนตารางที่มีอยู่แล้ว ไม่ต้องสร้างเพิ่ม:
- `khokai_pos_orders` — ออเดอร์ (กรอง `order_module='partner_pos'`) รายละเอียดอยู่ในคอลัมน์ `payload` (jsonb), รายการสินค้าอ้างด้วย `option_id`
- `product_options` — ชื่อสินค้า (`pick_name`) ตาม `option_id`
- `khokai_order_boxes` — วันจัดส่งรายลัง (`ship_date`)

ขั้นตอนที่แท็บ **🔗 Partner POS**:
1. **🔄 โหลดออเดอร์จาก Partner POS** — ดึงออเดอร์ partner_pos ที่มีสินค้ามาแสดง (เลขที่ / ลูกค้า / วันส่งปัจจุบัน / รายการ)
2. **จับคู่สินค้า** — สินค้าใน POS ↔ สินค้าใน MRP เพราะ MRP เก็บค่ากำลังผลิต/BOM ของตัวเอง · ชื่อใน POS เป็นภาษาเกาหลี (เช่น `돼지볼 1kg`) ส่วนใน MRP เป็นรหัส (เช่น `1호 1kg`) จึง **ต้องจับคู่เองครั้งแรก** (ระบบจำ `option_id` ไว้ให้ครั้งต่อไป) · เลือก "— ไม่นำเข้า —" เพื่อข้ามสินค้าที่ไม่ต้องการ
3. **⬇️ นำเข้าออเดอร์ที่เลือก** — สร้างออเดอร์ใน MRP (จำนวน = `qty` จาก POS, กำหนดส่ง = วันจัดส่งปัจจุบันของ POS ถ้ามี) แล้วไปดูผลที่แท็บ *ผลการวางแผน*
4. **☁️ เขียนวันส่งกลับ Partner POS** — เขียน "วันส่งได้จริง" (วันที่สินค้าทุกตัวในออเดอร์ผลิตเสร็จ = วันช้าสุด) กลับไปที่ `ship_date` ของออเดอร์นั้นใน POS

> **ปลอดภัยต่อข้อมูล POS:** การเขียนกลับจับคู่ด้วย `id` ของออเดอร์ แล้ว `update` เฉพาะ `payload.ship_date` (อ่านมาก่อน-แก้เฉพาะ ship_date-เขียนกลับ ไม่แตะรายการสินค้า/ยอดเงิน/สถานะ) และอัปเดต `ship_date` ในตาราง `khokai_order_boxes` เพื่อให้ POS แสดงทันที (mirror หลัก data-ownership ของ khokai-oms)
>
> ⚠️ Partner POS จะ **ลบ-แล้วสร้างลังใหม่** ในตาราง `khokai_order_boxes` ทุกครั้งที่แก้/แยกลัง — ถ้าแก้ลังใน POS *หลัง* เขียนวันส่งกลับ ค่าที่เขียนในตารางลังอาจหาย (แต่ `payload.ship_date` ยังอยู่) — ให้กดเขียนกลับซ้ำหลังจัดลังเสร็จ

## การใช้งาน
1. แท็บ **สินค้า & กำลังผลิต** — ตั้งค่าสินค้า สต็อก และกำลังผลิต
2. แท็บ **วัตถุดิบ & BOM** — คลังวัตถุดิบ + สูตรว่าสินค้า 1 ชิ้นใช้อะไรเท่าไร
3. แท็บ **ตั้งค่า** — วันเริ่มผลิต, วันทำงาน, เชื่อม Supabase
4. แท็บ **ออเดอร์** — โยนออเดอร์เข้าไป (หรือดึงจาก Partner POS ที่แท็บ 🔗)
5. แท็บ **ผลการวางแผน** — ดูผลลัพธ์ทั้งหมด

> กดปุ่ม **โหลดข้อมูลตัวอย่าง** เพื่อทดลองใช้ทันที
